## OneSync v0.1.4

本版本重点收紧 OneSync 管理面、Webhook 与镜像供应链安全，并补充 Unraid 升级和回滚说明。

### 更新内容

- 新增管理口令登录：使用随机 `ONESYNC_ADMIN_TOKEN` 建立 HttpOnly、SameSite 会话，所有管理 API 均需要认证。
- 为状态变更接口加入 CSRF、Origin、Host 与跨站请求校验，并增加登录和管理操作速率限制。
- 匿名健康检查仅返回存活状态与版本，登录后才显示同步、Graph 与账号详情。
- 关闭生产环境 `/docs`、`/redoc` 和 `/openapi.json`，并加入 CSP、防嵌入、禁止 MIME 嗅探等安全响应头。
- 管理端口改为显式绑定 Unraid LAN 地址；`ONESYNC_ALLOWED_HOSTS` 仅允许配置中的 IP 或域名。
- Webhook 收紧为 HTTPS 企业微信机器人地址，拒绝任意第三方地址、私网/回环/链路本地 DNS 解析、环境代理和 HTTP 重定向。
- Python 运行与开发依赖升级并使用带哈希的锁定文件；镜像构建移除运行阶段不需要的 pip、setuptools 和 wheel。
- 固定 OneDrive 基础镜像 digest，并新增精确依赖环境下的测试构建阶段。
- GitHub Actions 加入 Gitleaks、Hadolint、Trivy、Semgrep、镜像漏洞门禁，以及 SBOM、provenance 和 Cosign 无密钥签名流程。
- 更新 Compose、Unraid XML 和中英文部署文档，补充管理口令、LAN 绑定、Host 白名单、备份与回滚步骤。

### 部署注意

这是一次需要更新容器配置的安全升级：

1. 升级前备份 `/mnt/user/appdata/onesync`、`/mnt/user/onedrive` 以及当前 XML 或 Compose 配置。
2. 使用 `openssl rand -hex 24` 生成独立管理口令，并设置 `ONESYNC_ADMIN_TOKEN`；口令必须至少 16 个字符。
3. 将实际访问地址加入 `ONESYNC_ALLOWED_HOSTS`。默认 Unraid 示例为 `192.168.2.21,127.0.0.1,localhost`。
4. Compose 用户需设置 `ONESYNC_BIND_ADDRESS`；Unraid DockerMan 用户需确认 XML 的 `ExtraParams` 与实际 Unraid LAN IP 一致，并且不要再添加第二个 `8098` Port 映射。
5. 不要把 `.env`、管理口令、Webhook key 或 `docker compose config` 输出提交到 Git。
6. 容器重启后内存会话会失效，需要重新登录。通过 HTTPS 反向代理访问时，应同时启用安全 Cookie 并限制来源。
7. 当前发布流水线仅构建 `linux/amd64` 镜像。
